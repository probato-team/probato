# Contribuindo com o Probato

Obrigado por considerar contribuir com o **Probato**.

O Probato é uma **proposta Open Source em evolução**, construída a partir de necessidades reais e que depende diretamente da participação da comunidade para amadurecer, se ajustar e ganhar relevância.

Este documento define diretrizes simples para facilitar a colaboração, mantendo um ambiente produtivo, respeitoso e sustentável.

---

## Tipos de contribuição

Existem várias formas de contribuir com o projeto:

* Relatar bugs ou comportamentos inesperados
* Sugerir melhorias ou novas ideias
* Propor ajustes de arquitetura ou API
* Contribuir com código
* Melhorar a documentação
* Compartilhar casos de uso reais

Todas as contribuições são bem-vindas.

---

## Abrindo issues

As issues são o principal canal de comunicação do projeto.

Você pode abrir uma issue para:

* Reportar bugs
* Tirar dúvidas
* Propor melhorias
* Discutir ideias
* Compartilhar feedback de uso

Ao abrir uma issue, sempre que possível, informe:

* Contexto do problema ou ideia
* O que você esperava
* O que aconteceu (se aplicável)
* Versão da biblioteca ou aplicação

Não é necessário seguir um template rígido. Clareza é mais importante que formalidade.

---

## Propondo mudanças

Antes de iniciar grandes alterações, recomenda-se:

* Abrir uma issue para discutir a proposta
* Explicar o problema que se deseja resolver
* Justificar a abordagem sugerida

Isso ajuda a alinhar expectativas e evita retrabalho.

---

## Contribuindo com código

Para contribuições de código:

1. Faça um fork do repositório
2. Crie uma branch a partir da branch principal
3. Implemente a alteração de forma incremental
4. Garanta que os testes existentes continuem passando
5. Abra um Pull Request com uma descrição clara

Pull Requests pequenos e focados são preferíveis a grandes alterações de uma só vez.

---

## Estilo e qualidade

Alguns princípios importantes:

* Priorize clareza e legibilidade
* Evite soluções excessivamente complexas
* Siga os padrões existentes no projeto
* Prefira mudanças incrementais

O objetivo é manter o projeto simples, sustentável e fácil de evoluir.

---

## Documentação

Melhorias na documentação são tão importantes quanto código.

Correções de texto, exemplos, ajustes de clareza e novas seções são contribuições extremamente valiosas.

---

## Processo de revisão

Todas as contribuições passam por revisão.

O processo busca:

* Garantir alinhamento com a proposta do projeto
* Avaliar impactos técnicos
* Manter coerência com a visão do Probato

Discussões são bem-vindas e fazem parte do processo.

---

## Código de conduta

Espera-se que todos os participantes mantenham um ambiente respeitoso e colaborativo.

Críticas devem ser construtivas. Divergências técnicas são naturais e saudáveis quando conduzidas com respeito.

Comportamentos abusivos, ofensivos ou desrespeitosos não serão tolerados.

---

## Considerações finais

O Probato não é uma solução fechada. Ele evolui a partir do uso real e da colaboração da comunidade.

Sua participação — seja com código, ideias ou feedback — é fundamental para o crescimento do projeto.

Obrigado por contribuir.
